CSS transform property

-Lets you translate, rotate, scale, and/or skew elements on page

-transform functions

-will be applied in the order written

-can move elements left, right, up, or down, does not affect flow of document

transform: translate(x,y) moves element x from left, y from top

tranform: translateX(45px) moves on the x axis

tranform: translateY(45px) moves on the y axis

Translation

-Allows you to move elements left, right, up, or down.

Scaling

-scale(x,y) scales element horizontally then vertically. If only one number is supplied, it will apply it to both, maintaining the aspect ratio.

-example: transform: scale(1.5, 0.25);

-Can also use just scaleX(x) or scaleY(y).

-scaled elements grow from the center (i.e. stay in the same place). You can change this with transform-origin property.

-worth noting: To declare multiple transformations, provide a space-separated list of transform functions.

-also remember, scaled elements do not impact document flow. Scaled inline-block emements will still act like block elements. Transform does not make document reflow.

Rotation

-rotate()

-generally declared in degrees, positive = clockwise, neg = counterclockwise

Skew

-skew(x,y)

-skews along the x,y axis. Eden's notes and questions: why would you ever use this on text? It's a total type crime!

Changing the Origin of the Transform

-transform-origin property. Has the same syntax as the background-position property and defaults to th center of the object.

-rotate an ellipse, for example, will need this to look like a spirograph design.

Choose your Ordering Carefully

-order matters. Affects appearance.

Transitions

-Steps to create a simple transition:

1. Declare the original state of the element; the default

2. Declare the final state of your transitioned element (like a :hover state)

3. Include the transition functions in your default using propertities like transition-property, transition-duration, transition-timing-function, and transition delay

-transition property: what should be transitioned. Anything with a midpoint can be transitioned.

-properties that can be transitioned: Mozilla Transitions Reference Page

-transition-duration property: how long it will take in s or ms

-transition-timing-function property: control the pace of the transition in granular detail. I.E. start slow then speed up.

-ease, linear, ease-in, ease-out, ease-in-out. You can also define your own cubic-bezier function.

-steps function with start or end is another way to control how the transiton happens.

-Transition Delay Property: introduce a delay before the transition begins.In ms or s. Ex: a 50ms delay on mouseover of an ad, to make sure it's not accidentally triggered.

-Negative delays are possible, too. Starts transition in mid-stream.

-Transition Shorthand Property: You can combine them on one line with shorthand syntax

-Multiple Transitions: can change many things at once

Animations

-Unlike transitions, you control each step of tehe animation via key frames

-Keyframe: a snapshot that defines a starting or ending point of a transition. CSS annimations allow us to have many keyframes along the way, while regular transitions only have start/end keyframes.

-@keyframes rule

each keyframe looks like its own nested CSS declaration block.

-Two key terms, "from" and "to"

Inside each keyframe are the properties you want to animate, along with the animated values.

-Can be specified in any order. It's the percentage, not the order, that determines the sequence of events

-Animation properties: animation-name property is used to attach animation to an element

-animation-duration property defines length of time in s or msâ€”timme it takes to go from 1% to 100%

-animation-timing-funciton: ldetermines how the animation will progress over its duration. Can use ease, linear, ease-in, ease-out, ease-in-out, cubic-bezier90, step-start, start-end, steps(number, direction).

-animation-iteration-count: lets you define how many times an animation will play. Usually integer, though decimals will let you stop animation part way through. Also can say infinite. If omitted, default is 1 (will play once).

-animation-direction: Normally goes from 0 to 100%, then jumps back to 0. REverse will make it start at 100% and move back to zero.

-animation-delay: miliseconds or seconds before it begins.

-animation-fill-mode: defines what happens after last part ends, before first part begins again. Default is none. Other options are forwards, backwards, both.

-Shorthand Animation Property: space-separated list of values for animation-name, animation-duration, animation-timing-function, animation-delay, animation-iteration-count, animation-direction, animation-fill-mode, and animation-play-state
