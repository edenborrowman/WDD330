THE data- ATTRIBUTE

-A way of embeddng data in a web page using custom attributes ignored by the browser.

-Only meant to be used by JS.

-Format: starts with data- and only uses lowercase letters. Also includes an optional string value.

-Example: data-powers data-rating etc

-Essentially it's HTML's "jawn."

-Each element has a dataset properto to access any data- attributes it contains.

WEB STORAGE

-Key-value store on client's computer like cookies, but fewer restrictions, easier, and more capacity

-Info is not shared with the server, doesn't expire, can usually store up to 5 gb

-Window object will have property of localStorage. Native object with several properties and methods

-Info saved as key-value pairs, must be a string

-Also sessionStorage object available, though data only saved for the session

-Syntax: localStorage.setItem('name', 'Walter White');

-to check it's working, close browser. Then open and: localStorage.getItem('name');

-Remove entry using removeItem method: localStorage.removeItem('name'); or delete localStorage.name

-storage event fired when value is saved to local storage. Has lots of prperties. We can use this!

-key tells us the key of the item that changed; self explanatory: newValue, oldVAlue, storageArea (is it local or session storage)

-Only strings. Might seem like a limitation, but using JSON not so. Can parse the data to get it back and usable. ex: superman = JSON.parse(localStorage.getItem('superman'));

GEOLOCATION

-Just like it sounds. Location of device.

-Privacy concerns (!); User must give permission first

-Property of navigator object (from sitepoint JS novice to ninja ch 9)

-has method of getCurrentPosition() that returns position object to a callback function, called youAreHere in example. Ex: navigator.geolocation.getCurrentPosition(youAreHere);

-Properties include coords, latitude, longitude.

-Also has position.speed, position.altitude, position.heading, position.timestamp

-Has properties that calculate ccuracy, too. position.accuracy referes to latitude and longitude. lower return value = more accurate

-clearWatch() method stops callback

Multimedia

-audio and video tags make it easy to play multimedia without things like flash. Also there is a media API to control playback.

-src attrigute points to the audio file.

-Any content inside the audio or video tag will only display if the browser doesn't support audio and video playback; hence can be an error message.

-controls attribute can be used to display browser's native controls.

-properties include play(), pause(), volume(), muted(), currentTime(), playbackRate(), loop(), duration(). Used like this: video.play()

-Several events fire with audio and video: play event, pause event, volumechange event, loadmetadata event.

THE WINDOW OBJECT

-The window object is the browser's global object for JS. All properties, functions are methods of it.

-The Browser Object Model (BOM), is teh collection of properries and methods with information about the browser and computer screen.

-Global variable are created without const, let, or var. can be accessed in all parts of the program.

-parseInt() and isNaN() are methods of the global object. Could be written like window.parseInt(4.2), but we usually omit the window.

-alert(), confirm(), and prompt() are methods of the window object, too.

-can be used to get info about a user's browser. Eden's notes: Uh, privacy?

-Navigator property retruns refrence to navigator object (info about browser being used). userAgent property returns browser, OS being used.

-window.location contains info about the URL of the current page. the href property returns the url as a string: ex: window.location.href. It's a read/write property so it can be changed by assignment.

-window.location.protocol returns protocol used.

-window.location.host returns a string desctibing the domain ot the current URL and port number (Eden's Notes: Again. PRIVACY? As a user...major yikes.)

-window.location.port returns string with port number

-window.location.hash, window.location.origin, window.loction.assign...and other terrifyingly intrusive things a website can find out about you.

THE BROWSER HISTORY

-window.history can access iinfo about previously visited pages in a current browsing session.

-window.history.length shows how many pages visited before current one.

-window.history.go() method can be used to go to a specific page where 0 is the current page. Can also navigate forwards and backwards with window.history.forward() and window.history.back()

THE DOCUMENT OBJECT

-The window object contains a document object. It has properties and methods to deal with the page loaded in the window.

-document.write() writes a string of text to a page. If the page is already loaded, it replaces the current document.

-can write HTML, too. ex: document.write('h1>Hello, world!/h1>');

-its use is highly discouraged since it mixes JS within an HTML document.

COOKIES

-Small files saved locally on a computer. They store info to be used between requests.

-CAn only save 4kb of data, 20 files per domain.

-Personalize a user's browsing experience, preferences, etc.

-Starting to be replaced by localStorage API.

Take the form of a text file with name/value pairs separated by semicolons. Ex:"name=Superman; hero=true; city=Metropolis"

Creating Cookies

-First, assign it to JS's "cookie jar" using document.cookie property: ex: document.cookie = 'name=Superman';

-Can continue to apend data to end of string by adding more cookies and assigning them to document.cookie.

READING COOKIES

-To see content of cookie jar, just enter document.cookie.

-The split method breaks the string into an array containing each name/value pair. You use a "for of" loop to iterate through the array. Example: onst cookies = document.cookie.split("; "); for (crumb of cookies){ const [key,value] = crumb.split("="); console.log(`The value of ${key} is ${value}`); }

-To see an example of cookies used in the wild, you can visit almost any website, open the browser console, and type document.cookie .

COOKIE EXPIRY DATES

-Cookie are deleted once a browser session is finished, but they can be made persistent by adding "; expires=date" to the end of cookie when it's set where date is a value in the UTC string format.

THE PATH AND DOMAIN OF COOKIES

-Path can be changed so any page in the root directory can read the cookie. Done by adding ; path-/ to the end of the cookie when set.

-Can also set domain by adding '; domain-domainName" to end of cookie.

SECURE COOKIES

Adding the string ; secure will make sure it's only transmitted over secure HTTPS network.

DELETING COOKIES

-To remove a cookie, you need to set it to expire at a time in the past.
